## Part 1. a)

### 1. Definition of $M$-way Search Trees

An **$M$-way Search Tree** is a tree structure where each node can have a maximum of $M$ children and $M-1$ keys. It generalizes the binary search tree ($2$-way search tree). Specifically, a node containing $k$ children ($k \le M$) houses exactly $k-1$ keys sorted in ascending order: $K_1 < K_2 < \dots < K_{k-1}$.

These keys define boundary ranges for its subtrees such that:

- The first subtree contains only keys less than $K_1$.
    
- The $i$-th subtree contains keys strictly between $K_{i-1}$ and $K_i$.
    
- The last ($k$-th) subtree contains only keys greater than $K_{k-1}$.
    

### 2. Definition of a B-tree

A **B-tree** is a self-balancing, optimized $M$-way search tree designed for high-throughput block access in databases and file systems. A B-tree of order $M$ satisfies the following strict structural constraints:

- Every node contains at most $M$ children.
- Every internal node (except the root) must have at least $\lceil M/2 \rceil$ children.
- The root node has at least $2$ children if it is not a leaf node.
- A non-leaf node with $k$ children contains exactly $k-1$ keys.
- **Perfect Balance:** All leaf nodes must exist at the exact same depth/level.

### 3. Tree Analysis (Figure 1)

Given the B-tree of order $M = 3$ (Max keys = $2$, Min keys = $1$ except root):

- **Root:** `[15 | 19]`
    
- **Children:** Left: `[11 | 13]`, Middle: `[17]`, Right: `[21]`
    

#### **How many keys must be inserted to increase the height by one level?**

- **Answer:** **1 key**
    
- **Explanation:** A B-tree's height increases strictly when its root node splits. The current root `[15 | 19]` is completely full. If we insert any key that falls into the already full left child `[11 | 13]` (e.g., key `12`), that child overflows to `[11 | 12 | 13]` and splits. The median key `12` is pushed up into the root. This causes the root to overflow (`[12 | 15 | 19]`), forcing it to split and push the median key `15` upward to form a new single-element root, thereby increasing the height by $1$.
    

#### **How many keys must be deleted to reduce the height by one level?**

- **Answer:** **4 keys**
    
- **Explanation:** The current tree has a total of $6$ keys distributed across $2$ levels. To shrink the tree to a single level (height = 1), all remaining keys must reside exclusively within the single root node. Since $M = 3$, a single node can hold at most $M - 1 = 2$ keys. To reduce the total number of keys from $6$ down to a maximum of $2$, we must delete exactly $6 - 2 = 4$ keys.
    

## Part 1. b)

### 1. Construction of B+ tree of Order 5

- **Constraints for Order 5:** Max keys in any node = $4$. Min keys in leaf node = $\lceil (5-1)/2 \rceil = 2$.
- When a leaf node splits with $5$ keys, the first $3$ keys remain on the left, the remaining $2$ keys go to the right, and the smallest key of the right child is _copied_ up to the parent.
    

#### **Step-by-step Insertion Sequence:** `16, 18, 14, 20, 12, 22, 10, 24, 8, 6, 15, 11, 26, 30`

1. **Insert 16, 18, 14, 20:**
    
    $$\text{Leaf: } [14, 16, 18, 20]$$
    
2. **Insert 12:** Node overflows $\rightarrow$ Splits.
    
    $$\text{Root: } [18]$$
    
    $$\text{Leaves: } [12, 14, 16] \rightarrow [18, 20]$$
    
3. **Insert 22, 10, 24:**
    
    $$\text{Root: } [18]$$
    
    $$\text{Leaves: } [10, 12, 14, 16] \rightarrow [18, 20, 22, 24]$$
    
4. **Insert 8:** Left leaf overflows $\rightarrow$ Splits; `14` is copied up.
    
    $$\text{Root: } [14, 18]$$
    
    $$\text{Leaves: } [8, 10, 12] \rightarrow [14, 16] \rightarrow [18, 20, 22, 24]$$
    
5. **Insert 6, 15:**
    
    $$\text{Root: } [14, 18]$$
    
    $$\text{Leaves: } [6, 8, 10, 12] \rightarrow [14, 15, 16] \rightarrow [18, 20, 22, 24]$$
    
6. **Insert 11:** First leaf overflows $\rightarrow$ Splits; `11` is copied up.
    
    $$\text{Root: } [11, 14, 18]$$
    
    $$\text{Leaves: } [6, 8, 10] \rightarrow [11, 12] \rightarrow [14, 15, 16] \rightarrow [18, 20, 22, 24]$$
    
7. **Insert 26:** Last leaf overflows $\rightarrow$ Splits; `24` is copied up.
    
    $$\text{Root: } [11, 14, 18, 24]$$
    
    $$\text{Leaves: } [6, 8, 10] \rightarrow [11, 12] \rightarrow [14, 15, 16] \rightarrow [18, 20, 22] \rightarrow [24, 26]$$
    
8. **Insert 30:**
    
    $$\text{Root: } [11, 14, 18, 24]$$
    
    $$\text{Leaves: } [6, 8, 10] \rightarrow [11, 12] \rightarrow [14, 15, 16] \rightarrow [18, 20, 22] \rightarrow [24, 26, 30]$$
    

### 2. Deletion Steps

#### **Step 1: Delete 16**

- Key `16` is located in leaf `[14, 15, 16]`.
    
- After removal, the leaf becomes `[14, 15]`. Since it retains $2$ keys ($\ge \text{minimum requirement}$), no structure adjustment is needed.
    

#### **Step 2: Delete 8**

- Key `8` is removed from the first leaf `[6, 8, 10]`, leaving `[6, 10]`. It satisfies the minimum key constraint, so no structural modification is required.
    

#### **Step 3: Delete 15**

- Key `15` is removed from leaf `[14, 15]`, leaving only `[14]`. This underflows (less than $2$ keys).
    
- **Borrowing Strategy:** Its left sibling `[11, 12]` is at minimum capacity ($2$ keys), but its right sibling `[18, 20, 22]` has $3$ keys.
    
- We borrow the lowest key (`18`) from the right sibling. Leaf $L_3$ becomes `[14, 18]`, and the right sibling becomes `[20, 22]`.
    
- The parent element guiding to the right child updates its separator index to the new lowest key (`20`).
    

#### **Final Resultant B+ Tree:**

```
               [ 11  |  14  |  20  |  24 ]
              /      |      |      |      \
     [6, 10] ---> [11, 12] ---> [14, 18] ---> [20, 22] ---> [24, 26, 30]
```

## Part 1. c)

### 1. What is an AVL tree?

An **AVL Tree** is a self-balancing Binary Search Tree (BST) where the height difference between the left and right subtrees of **any** node (called the **Balance Factor**) is strictly restricted to $\{-1, 0, 1\}$.

$$\text{Balance Factor (BF)} = \text{Height}(\text{Left Subtree}) - \text{Height}(\text{Right Subtree})$$

### 2. Types of Rotations

Four structural rotations restore balance when insertions or deletions cause a node's $\text{BF}$ to become $\pm 2$:

1. **LL Rotation (Single Right Rotation):** Applied when an imbalance occurs in the left subtree of the left child.
    
2. **RR Rotation (Single Left Rotation):** Applied when an imbalance occurs in the right subtree of the right child.
    
3. **LR Rotation (Double Rotation):** Applied when an imbalance occurs in the right subtree of the left child. Consists of an RR rotation followed by an LL rotation.
    
4. **RL Rotation (Double Rotation):** Applied when an imbalance occurs in the left subtree of the right child. Consists of an LL rotation followed by an RR rotation.
    

### 3. Height of an AVL Tree

For an AVL tree containing $n$ nodes, the maximum upper bound height $h$ behaves logarithmically:

$$h \le 1.44 \log_2(n + 2) - 0.328 = O(\log n)$$

### 4. Construction Process

Keys: `15, 19, 21, 13, 10, 23, 22, 5, 7`

- **Insert 15, 19, 21:** Triggers an **RR imbalance** at node `15`. Left rotation on `15` yields:
    

```
    19
   /  \
  15  21
```

- **Insert 13, 10:** Triggers an **LL imbalance** at node `15`. Right rotation on `15` yields:
    

```
    19
   /  \
  13  21
 /  \
10  15
```

- **Insert 23, 22:** Triggers an **RL imbalance** at node `21`. Double rotation yields:
    

```
    19
   /  \
  13  22
 / \  / \
10 15 21 23
```

- **Insert 5, 7:** Triggers an **LR imbalance** at node `10`. Double rotation yields the final constructed tree:
    

```
        19
       /  \
      13   22
     / \   / \
    7  15 21  23
   / \
  5  10
```

### 5. Deletion Steps

#### **Delete 22:**

An internal node. Replace it with its inorder successor (`23`).

```
        19
       /  \
      13   23
     / \   /
    7  15 21
   / \
  5  10
```

#### **Delete 21:**

Removing leaf `21` causes root node `19` to underflow ($\text{BF} = +2$). Since its left child `13` has $\text{BF} = +1$, this triggers an **LL Case**. Perform a Right Rotation on node `19`:

```
        13
       /  \
      7    19
     / \   / \
    5  10 15  23
```

#### **Delete 13:**

Replace the root node with its inorder successor (`15`).

```
        15
       /  \
      7    19
     / \     \
    5  10     23
```

#### **Delete 10:**

Removing leaf node `10` leaves the tree perfectly balanced.

#### **Final Resultant AVL Tree:**

```
        15
       /  \
      7    19
     /       \
    5         23
```
