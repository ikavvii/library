These questions test theoretical principles of testing and the practical application of test case design.

- **Testing Principles:**
    - Defend or comment on the statement: **"Exhaustive Testing is Impossible"** with appropriate examples.
    - Explain the significance of the **Pareto Principle** in software testing.
    - Describe the background of the **"Defect Clustering"** principle.
    - Explain the significance of the **Cyclomatic Complexity Constant**.
- **White Box / Path Testing:**
    - Given a logic for **Banking Interest Rates** (based on deposit amount and period), write pseudo-code and apply **Path Testing**.
    - Apply **Basis Path Testing** to determine the number of test cases and generate a report for:
        - A code snippet finding the **largest of two numbers**.
        - A program that reads numbers and identifies them as **positive, negative, or zero** until the user quits.
- **Black Box Testing:**
    - Design test cases for a **Quadratic Solver** function ($ax^2 + bx + c = 0$) and tabulate the results in standard format.
    - Apply Black Box methods to test a module for **customer authentication** in ATM Banking.
    - Design test cases for a **Next Date program** (reading DD/MM/YYYY) using **Boundary Value Analysis (BVA)** and **Robust Testing**.
