### **Question 3 (25 Marks Total)**

#### **Extracted Questions**

- **Subdivision A (3 Marks):** Assess the usefulness and values of JavaScript in all stages of Full Stack based Enterprise Applications.
- **Subdivision B (10 Marks):** Evaluate the comparative advantages of single-page applications (SPA) versus multipage applications (MPA) in web development, with a specific emphasis on their impact on user experience and development efficiency.
- **Subdivision C (12 Marks):** Compare and contrast the use of MERN stack and MEAN Stack for the development of "Bookface," a social networking site for book lovers. The site requires registration, CAPTCHA verification, and real-time interaction for book comments.

#### **Detailed Solutions**

**A. JavaScript Across the Full Stack [3 Marks]** JavaScript is unique because it serves as a **universal language** for every layer of an enterprise application.

- **Client-Side:** Enables highly interactive and responsive user interfaces through frameworks like React or Angular.
- **Server-Side:** Using **Node.js**, JavaScript handles backend logic, allowing developers to use a single language for the entire stack, which reduces context switching and simplifies development.
- **Database:** Many NoSQL databases, such as **MongoDB**, use JavaScript-like syntax for querying and data manipulation, providing a seamless data-enabling layer.

**B. SPA vs. MPA: UX and Efficiency [10 Marks]**

- **Single-Page Application (SPA):**
    - **User Experience (UX):** Provides a fluid, **app-like experience**. Because content is updated dynamically via AJAX/Fetch without full page refreshes, transitions are seamless and latency is minimized during interaction.
    - **Development Efficiency:** Encourages a **component-based architecture**, allowing for high code reusability. However, it requires more complex client-side state management and initial configuration for SEO.
- **Multi-Page Application (MPA):**
    - **User Experience (UX):** Traditional navigation where each action requests a new page from the server. This can lead to visible lag and a less responsive feel compared to SPAs.
    - **Development Efficiency:** Generally easier for SEO and initial page loads. It is simpler to develop for content-heavy sites but becomes cumbersome for highly interactive, real-time features.

**C. MERN vs. MEAN for "Bookface" [12 Marks]** For a social platform like "Bookface" that requires high interactivity (commenting, author/book browsing) and real-time updates:

- **MERN (React):** Utilizes the **Virtual DOM**, which is highly efficient for the frequent, dynamic updates typical of social media comments and likes. Its component-based nature allows developers to build modular UI elements for books and authors that can be reused across the site.
- **MEAN (Angular):** Provides a comprehensive, prescriptive framework. Its **two-way data binding** is powerful but can become performance-heavy in applications with massive amounts of real-time data.
- **Implementation Note:** Both stacks easily support security measures like **CAPTCHA** during registration to prevent webbot access, ensuring the integrity of the user base.

### **Question 4 (25 Marks Total)**

#### **Extracted Questions**

- **Subdivision A (3 Marks):** Compare and contrast APIs and Micro Services with respect to inter-enterprise applications.
- **Subdivision B (10 Marks):** Analyze a case study for an online cab service (managing employees, drivers, vehicles, and reservations) for the use of SOAP and REST based enterprise applications.
- **Subdivision C (12 Marks - Option i):** Outline the various step-by-step tasks with syntax and semantics for the Design, Creation, Deploying, Testing, and Consuming of REST Web services for a travel agency website.
- **Subdivision C (12 Marks - Option ii):** 1) Analyze all Architectural Constraints with their benefits in REST. 2) Analyze and discuss the various standard stacks used in SOAP (SOAP, WSDL, UDDI).

#### **Detailed Solutions**

**A. APIs vs. Microservices [3 Marks]**

- **API:** Acts as the **interface** or contract that allows two different systems to communicate. It defines _how_ requests and responses should look.
- **Microservices:** An **architectural style** that breaks a large application into small, independent services. Each service performs a single function and communicates with others via APIs.
- **Enterprise Role:** APIs are the "doorways" for inter-enterprise integration, while Microservices provide the internal modularity that makes those integrations easier to manage.

**B. Case Study: SOAP and REST for Online Cab Service [10 Marks]**

- **SOAP Usage:** Recommended for **internal secure data exchange**, such as managing driver license records, employee salary rates, and insurance information. Its strict standards and ACID compliance ensure the reliability of sensitive administrative data.
- **REST Usage:** Ideal for **customer-facing interactions** and third-party integrations. This includes mobile reservations, real-time vehicle tracking, and integration with Map Services. REST’s use of **JSON** makes it lightweight and faster for mobile users who need to check cab availability on the go.

**C. Option (i): REST Web Service Implementation Lifecycle [12 Marks]**

1. **Design:** Identify resources (e.g., `/reservations`, `/vehicles`) and map them to HTTP methods (POST to create, GET to view).
2. **Creation:** Implement the logic using a framework like JAX-RS or Node.js/Express, defining the payload structure (typically JSON).
3. **Deploying:** Host the service on a web server or cloud platform where it can be accessed via a stable URI.
4. **Testing:** Use tools like Postman to verify that the endpoints return the correct status codes and data.
5. **Consuming:** The frontend (React/Angular) or external partners use AJAX or Fetch to call the service and integrate the data into their UI.

**C. Option (ii): REST Constraints and the SOAP Stack [12 Marks]**

- **REST Constraints:**
    - **Statelessness:** Improves scalability as the server doesn't store client sessions.
    - **Uniform Interface:** Simplifies architecture by using standard HTTP verbs, making it platform-agnostic.
    - **Layered System:** Allows for intermediaries like load balancers and caches, enhancing security and performance.
- **SOAP Standard Stack:**
    - **SOAP:** The messaging protocol.
    - **WSDL (Web Services Description Language):** The XML-based contract describing what the service does.
    - **UDDI (Universal Description, Discovery, and Integration):** The directory used to find web services.
    - **Benefit:** This stack ensures **strict interoperability** in cross-enterprise supply chains where systems run on different platforms and programming languages.
